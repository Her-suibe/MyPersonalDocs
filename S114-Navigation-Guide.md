# S114 学习导航手册（Student Navigation Guide）
版本：v1.0  
适用对象：S114 AI 语校全体学生  

---

## 0. 欢迎加入 S114！

S114 不是一门传统的“软件操作课”，而是一门围绕 AI × GitHub × VS Code 的“未来工作方式”实训课。

这门课的目标不是把你变成程序员，而是帮助你建立一种在论文、项目、工作中都能用到的能力：

用 AI 辅助思考与写作，用 VS Code 结构化整理内容，用 GitHub 管理版本与小组协作。

本手册的目的只有一个：让你在这门课里不迷路。

---

## 1. 课程目标：你最终会获得什么？

这门课结束时，你不一定记住所有操作细节，但应该能做到：

1. 用 AI 辅助思考与写作  
2. 用 Markdown 组织报告、论文和资料  
3. 用 GitHub 管理版本与小组协作  
4. 用 VS Code 作为统一的写作与工作空间  
5. 完成一个可展示的成果（小论文、项目报告、作品集等）

这些能力会在你的未来课程论文、商业分析、求职作品集等场景中持续发挥作用。

---

## 2. 工具链导航：这门课的“地图”

理解 AI、VS Code、GitHub 的关系非常重要：

```
AI（ChatGPT / Copilot / Gemini 等）
          ↓
VS Code（写内容、做结构、打草稿）
          ↓
GitHub（保存版本、团队协作、对外展示）
```

简单理解为：

- AI：帮你“想和改”
- VS Code：帮你“写和整理”
- GitHub：帮你“存和合作”

---

## 3. 学习路径：本学期的路线图（4 个阶段）

整个课程不会要求你一次学会所有工具，而是按以下阶段逐步推进。

### Phase 1：能用（Install & Login）
目标：装好所有软件，能打开并登录即可。

### Phase 2：能写（AI × Markdown）
目标：在 AI 的帮助下写出结构清晰的内容，并用 Markdown 管理。

### Phase 3：能管（GitHub × 版本管理）
目标：把文档从“散落 everywhere”变成“集中管理，有版本记录”。

### Phase 4：能协作（小组协作 × PR）
目标：和组员一起完成一次真实的 GitHub 协作流程。

---

## 4. 每周只需要做到的 4 件“小事”

1. 用 AI 总结一段材料（5–10 分钟）  
2. 写一篇 200 字 Markdown 学习笔记  
3. 把笔记 push 到 GitHub（一次 commit）  
4. 在组内 Issue 区做一次学习更新  

做到这 4 点，你的整体进度就会领先大部分人。

---

## 5. 常见名词导航（不用背，只要理解）

- Markdown：更干净简单的写作格式  
- Repo：你的项目空间  
- Clone：把 Repo 下载到你的电脑  
- Commit：保存一次修改  
- Push：把修改上传到 GitHub  
- Pull：拉取别人的更新  
- Issue：任务与讨论区  
- PR（Pull Request）：提交改动、协作流程  
- VS Code：你的统一工作台（写作 + AI + Git）

---

## 6. 学习中遇到问题怎么办？

推荐解决顺序：

1. 先问 AI（提供报错、截图、描述问题）  
2. 问组员  
3. 在课程 Repo 的 Issue 区提问  
4. 联系助教处理账号/权限问题  

尽量不要沉默。如果你不提问，你的卡点别人永远不知道。

---

## 7. 本课程的最终成果（你会留下什么？）

课程结束时，你至少会留下：

- 一个 GitHub 项目（含你的学习笔记与 Markdown 文档）
- 一组 commit 历史记录（你的创作过程）
- 至少一次协作记录（Issue、PR 或评论）
- 一个可继续扩展的个人项目或作品集

这些内容未来可以放进简历或求职作品集中。

---

## 8. 给你的三条学习建议

1. 不要害怕出错（GitHub 可回退，不会“炸掉”项目）  
2. 把 AI 当“合作者”，不是“替身”  
3. 不需要一次学会所有内容，按阶段推进即可  

---

## 9. 你的第一步任务（约 5 分钟）

1. 打开课程 GitHub 仓库，进入你的小组目录  
2. 新建一个 Markdown 文件：`my-first-note.md`  
3. 写一句话：  
   “我希望在 S114 课程中学到的内容是：……”  
4. commit + push  
5. 在小组 Issue 区发一个打卡帖：  
   标题示例：`[Week X] 学习打卡 - 你的姓名`  
   内容示例：  
   “我已经完成了第一份 Markdown 文件，并成功 push 到 Repo。”  

完成这一组动作后，你已经正式踏入 AI × GitHub 的工作方式世界。

欢迎加入 S114。
